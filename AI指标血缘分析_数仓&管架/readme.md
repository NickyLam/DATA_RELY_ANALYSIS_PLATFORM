1.管架脚本目录存放的是管架各层系统脚本，数仓脚本存放的是数仓各层脚本。
2.脚本路径下均包含ddl和dml文件夹，ddl为建表脚本，dml为数据跑批脚本，管架脚本下的ext文件夹，为抽数脚本的路径。
3.数仓脚本-数仓视图逻辑文档，登记的是数仓ORACLE数据库内建立的视图，来源为oracle系统表all_views，因财务集市跟数仓均属于仓内系统，所以会使用到视图交互。
4.目前数仓、财务集市、管架的供数路线主要为数仓->财务集市（通过视图供数）->管架（通过卸数文件供数），但是也存在少量通过数仓->管架（通过卸数文件供数）的供数链路。
5.财务集市、管架的数据来源系统大部分都是数仓来，少量不通过数仓，例如财务集市可能会从风险集市拿数据，管架可能会从行内业务系统直接拿数据，此种情况为少数，本次血缘分析可先忽略。
6.管架指标表名为mc_index_fact，该表存在多组sql脚本加工。
