## 血缘链路分析

### 核心血缘链路

以下为监管报送系统的核心血缘链路：

#### 1. 财务模块链路

```
oracle:finance:transaction_log
    └── oracle:finance:account_balance
    └── oracle:finance:daily_report
        └── oracle:finance:monthly_summary
```

**链路特点**：
- 交易流水 -> 账户余额：实时同步
- 交易流水 -> 日报汇总：日终批处理
- 日报 -> 月报：月度汇总

#### 2. 跨系统链路（Oracle -> TDH）

```
oracle:finance:transaction_log
    └── tdh:datalake:raw_transaction
        └── tdh:datalake:cleaned_transaction
            └── tdh:datalake:aggregated_report
                └── oceanbase:report:daily_metrics
                    └── oceanbase:report:kpi_dashboard
```

**链路特点**：
- 数据从 Oracle 实时抽取到 TDH 数据湖
- 经过清洗、聚合后写入 OceanBase 报表库
- 最终汇总到 KPI 仪表盘

#### 3. CRM 客户链路

```
oracle:crm:customer_info
    └── tdh:datalake:customer_profile
        └── gbase:analytics:user_behavior
            └── gbase:analytics:customer_segment
```

**链路特点**：
- 客户主数据流入数据湖
- 生成客户画像
- 用于用户行为分析和客户分群

## 问题与建议

### 发现的问题

| 问题类型 | 描述 | 影响 | 建议 |
|----------|------|------|------|
| 动态 SQL | 存储过程包含动态 SQL | 血缘解析不完整 | 增加运行时采集辅助 |
| 复杂表达式 | 字段转换表达式复杂 | 准确率下降 | 增加人工标注 |
| 跨库关联 | 多库关联查询 | 血缘链路断裂 | 增强跨库解析 |

### 优化建议

1. **解析引擎增强**
   - 增加 PL/SQL 动态 SQL 解析能力
   - 支持更多 Oracle 特有语法
   - 优化复杂表达式解析

2. **运行时采集**
   - 开启 Oracle 审计日志采集
   - 配置 V$SQL 视图采集
   - 实现静态与运行态血缘融合

3. **人工标注补充**
   - 对复杂链路增加人工标注
   - 定期验证标注准确性
   - 建立标注审核流程
