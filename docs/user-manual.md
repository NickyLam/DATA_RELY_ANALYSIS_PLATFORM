## 1. 平台概述

### 1.1 平台简介

数据血缘分析平台是一个用于追踪和分析数据流转关系的系统。它可以帮助您：

- **追溯数据来源**：了解数据的原始出处
- **分析数据流向**：追踪数据的加工和流转路径
- **评估变更影响**：分析数据变更对下游的影响
- **排查数据问题**：快速定位数据异常的根源

### 1.2 主要功能

| 功能模块 | 功能描述 | 适用场景 |
|----------|----------|----------|
| 资产搜索 | 搜索表、字段等数据资产 | 快速定位目标数据 |
| 血缘查询 | 查看表的上下游血缘关系 | 了解数据来源和去向 |
| 字段级血缘 | 查看字段的转换链路 | 分析字段计算逻辑 |
| 问题排查 | 关联运行态信息排查问题 | 定位数据质量问题 |

### 1.3 系统架构

平台采用前后端分离架构，包含以下组件：

- **前端界面**：基于 React 的 Web 应用
- **后端服务**：基于 FastAPI 的 API 服务
- **图数据库**：Neo4j 存储血缘关系
- **元数据库**：PostgreSQL 存储元数据
- **缓存服务**：Redis 加速查询响应

## 3. 资产搜索

### 3.1 功能说明

资产搜索模块用于快速定位目标数据资产（表、字段等）。

### 3.2 搜索步骤

#### 基本搜索

1. 在顶部搜索框输入关键词
2. 点击搜索按钮或按 Enter 键
3. 系统返回匹配的资产列表

#### 高级搜索

1. 点击搜索页面右侧的「高级筛选」按钮
2. 选择筛选条件：
   - **数据源**：选择特定数据库
   - **业务域**：选择业务分类
   - **表类型**：选择表/视图
3. 点击「应用筛选」执行搜索

### 3.3 搜索结果

搜索结果包含以下信息：

| 字段 | 说明 |
|------|------|
| 名称 | 资产名称 |
| 类型 | 表/视图/字段 |
| 数据源 | 所属数据库 |
| 业务域 | 所属业务分类 |
| 描述 | 资产描述信息 |

### 3.4 操作说明

- **查看详情**：点击资产名称进入详情页
- **查看血缘**：点击「血缘」按钮进入血缘图
- **导出结果**：点击「导出」下载搜索结果

## 4. 血缘查询

### 4.1 功能说明

血缘查询模块展示数据表的上下游流转关系，以图谱形式可视化。

### 4.2 进入血缘图

1. 通过搜索找到目标表
2. 点击表名称进入详情页
3. 点击「查看血缘」按钮
4. 进入血缘图谱页面

### 4.3 血缘图谱界面

血缘图谱包含以下元素：

| 元素 | 说明 |
|------|------|
| 节点 | 表示数据表 |
| 连线 | 表示血缘关系（数据流向） |
| 箭头 | 表示数据流转方向 |
| 标签 | 显示表名称 |

### 4.4 图谱操作

#### 方向切换

- **上游血缘**：显示数据的来源（追溯）
- **下游血缘**：显示数据的去向（影响）
- **双向血缘**：同时显示上下游

#### 深度调整

- 使用深度滑块调整显示层级
- 默认深度为 5 层
- 最大支持 10 层深度

#### 节点交互

- **点击节点**：查看表详情
- **双击节点**：以该节点为中心重新展开
- **拖拽节点**：调整节点位置
- **缩放图谱**：使用鼠标滚轮或工具栏

### 4.5 导出功能

- **导出 PNG**：导出图谱图片
- **导出 JSON**：导出血缘关系数据

## 5. 字段级血缘

### 5.1 功能说明

字段级血缘展示字段之间的转换链路，帮助理解字段的计算逻辑。

### 5.2 查看字段血缘

1. 进入表详情页
2. 找到目标字段
3. 点击字段名称或「查看链路」按钮
4. 进入字段血缘页面

### 5.3 字段链路界面

字段链路页面包含：

| 区域 | 内容 |
|------|------|
| 链路图 | 字段的转换路径 |
| 表达式列表 | 转换表达式详情 |
| 源字段 | 数据来源字段 |
| 目标字段 | 最终字段 |

### 5.4 理解转换表达式

转换表达式展示了字段如何从源字段计算而来：

| 表达式类型 | 说明 | 示例 |
|------------|------|------|
| 直接映射 | 无转换 | `source.field -> target.field` |
| 算术运算 | 数值计算 | `a + b` |
| 函数转换 | 函数处理 | `UPPER(name)` |
| 条件分支 | 条件判断 | `CASE WHEN...` |

## 6. 问题排查

### 6.1 功能说明

问题排查模块关联血缘信息与运行态信息，帮助定位数据问题。

### 6.2 进入问题排查

1. 进入表详情页
2. 点击「问题排查」标签
3. 查看相关信息

### 6.3 排查信息

问题排查页面包含：

| 信息类型 | 内容 | 用途 |
|----------|------|------|
| 最近批次 | 最近执行的作业批次 | 了解处理时间 |
| 变更时间线 | 表结构和数据的变更记录 | 定位变更影响 |
| 上游状态 | 上游表的运行状态 | 检查数据来源 |
| 下游影响 | 下游表受影响情况 | 评估问题范围 |

### 6.4 排查流程

**步骤 1：确认问题**
- 检查最近批次执行状态
- 查看数据质量指标

**步骤 2：追溯来源**
- 检查上游表状态
- 查看上游批次执行情况

**步骤 3：定位原因**
- 对比变更时间线
- 检查异常时间点

**步骤 4：评估影响**
- 查看下游影响范围
- 通知相关人员

## 7. 常见问题

### 7.1 登录问题

**Q: 无法登录系统？**

A: 请检查以下问题：
1. 用户名和密码是否正确
2. 网络连接是否正常
3. 账户是否被锁定
4. 如仍无法登录，请联系管理员

### 7.2 搜索问题

**Q: 搜索结果为空？**

A: 可能原因：
1. 关键词不匹配任何资产
2. 数据尚未采集
3. 筛选条件过于严格
4. 尝试使用更通用的关键词

**Q: 搜索响应慢？**

A: 可能原因：
1. 搜索词过于复杂
2. 数据量较大
3. 网络延迟
4. 稍后重试或联系管理员

### 7.3 血缘问题

**Q: 血缘图谱显示不完整？**

A: 可能原因：
1. 血缘深度设置过小
2. 数据采集尚未完成
3. 部分表未被纳入系统
4. 增加深度或等待采集完成

**Q: 血缘关系不准确？**

A: 可能原因：
1. SQL 解析存在偏差
2. 动态 SQL 难以解析
3. 请通过人工标注补充

### 7.4 性能问题

**Q: 页面加载慢？**

A: 建议：
1. 清理浏览器缓存
2. 减少血缘深度
3. 使用筛选缩小范围
4. 刷新页面重试

## 8. 最佳实践

### 8.1 血缘查询最佳实践

1. **从已知表开始**
   - 从熟悉的业务表开始查询
   - 逐步扩展血缘范围

2. **合理设置深度**
   - 日常查询使用 3-5 层深度
   - 深度分析不超过 10 层

3. **使用筛选功能**
   - 过滤非相关表
   - 专注于目标数据域

4. **保存常用查询**
   - 记录常用表的血缘结果
   - 导出关键血缘关系

### 8.2 问题排查最佳实践

1. **建立排查流程**
   - 从问题表开始追溯
   - 按时间线排查变更

2. **记录排查结果**
   - 记录问题原因和解决方法
   - 归档常见问题案例

3. **及时通知相关方**
   - 发现问题及时通知上下游负责人
   - 协同处理数据问题

### 8.3 数据资产管理最佳实践

1. **定期检查血缘**
   - 定期验证核心表血缘准确性
   - 发现异常及时反馈

2. **维护元数据**
   - 补充表描述信息
   - 更新业务域分类

3. **参与人工标注**
   - 对关键血缘进行人工标注
   - 提升系统准确性
