# 路线图

语言版本：中文 | [English](ROADMAP.md)

这份路线图保持务实：优先提升项目作为开源数据血缘分析器的可评估性、可运行性、可扩展性和可信度。

## 当前：v2.2.x

- 完善中英文开源入口文档。
- 保持 demo 截图和 GIF 与前端界面一致。
- 扩展安全合成 SQL 示例，覆盖 Oracle、存储过程和数仓交换表工作流。
- 保持 Issue 模板、安全策略和敏感数据处理规范一致。

## 下一步：v2.3

- 增加可公开解析的最小样例数据集，不依赖私有企业数据。
- 增加一键 demo 模式，只加载内置安全样例 SQL。
- 在 Swagger/OpenAPI 文档中补充响应示例。
- 强化 CTAS、INSERT SELECT、MERGE、EXCHANGE PARTITION、存储过程等解析回归测试。
- 增加血缘和口径证据导出的示例文件。

## 后续：v2.4+

- 为血缘图输出增加明确的 graph schema version。
- 支持解析批次之间的血缘 diff。
- 增加可选容器化本地部署方式。
- 增加用户提交 SQL 片段和截图的可配置脱敏/遮罩能力。
- 改进前端可访问性、键盘导航和大图谱性能。

## 非目标

- 不提供托管用户私有 SQL 的公共服务。
- 不提交真实企业数据、本地缓存或生成数据库。
- 不让 `core/` 解析逻辑依赖 FastAPI 或 UI 框架。
