## 2026-05-13 修复：展示层三个 Bug（A/B/C）— 第二轮修正

### 第一轮修复失败原因分析
第一轮修复（见上文）没有解决根本问题：
- `_bare()` 函数只是简单取 `split('.')[-1]`，无法识别 `RRP_MDL.O_ICL_CMM_INDV_CUST_BASIC_INFO` 和 `ICL.CMM_INDV_CUST_BASIC_INFO` 是同一个表
- 导致 `node_map` key 不同、`edge_key` 不同、`fm_key` 不同，去重全部失效

### 正确根因
**`O_ICL_*` 命名规范**：`RRP_MDL.O_ICL_XXX` 是 `ICL.XXX` 的同义词/视图，Parser 同时识别到了两种形式。
- `_bare('ICL.CMM_INDV_CUST_BASIC_INFO')` → `CMM_INDV_CUST_BASIC_INFO`
- `_bare('RRP_MDL.O_ICL_CMM_INDV_CUST_BASIC_INFO')` → `O_ICL_CMM_INDV_CUST_BASIC_INFO`
- 两个裸表名不同，所有去重逻辑失效

### 第二轮修复

#### Bug A + C 修复：`api_server.py` 的 `_bare()` 函数（第 217-237 行）
```python
def _bare(table_name: str) -> str:
    parts = table_name.split(".")
    schema = parts[0].upper() if len(parts) > 1 else ""
    bare = parts[-1].upper()
    # RRP_MDL.O_ICL_* -> ICL_*
    if bare.startswith("O_ICL_"):
        bare = bare[2:]  # O_ICL_ -> ICL_
        return bare
    # ICL.XXX -> ICL_XXX（与 O_ICL_* 统一前缀）
    if schema == "ICL" and not bare.startswith("ICL_"):
        bare = f"ICL_{bare}"
    return bare
```
- 现在三个变体都映射为 `ICL_CMM_INDV_CUST_BASIC_INFO`，可以正确去重
- 同时修复了 Bug A（字段映射重复）和 Bug C（ICL 节点重复）

#### Bug B 修复：`lineage-graph.js` 的 `isQueryNode` 判断（第 222-227 行）
```javascript
const qtBare = qt.includes('.') ? qt.split('.').pop() : qt;
const srcBare = edge.source_table.includes('.') ? edge.source_table.split('.').pop() : edge.source_table;
const tgtBare = edge.target_table.includes('.') ? edge.target_table.split('.').pop() : edge.target_table;
const isQueryNode = edge.source_table === qt || edge.target_table === qt ||
                    srcBare === qtBare || tgtBare === qtBare;
```
- 同时比较完整表名和裸表名，解决带/不带 schema 的匹配问题
- 非查询边 opacity 保持 0.65（第一轮已修）
