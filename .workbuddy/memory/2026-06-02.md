## SOURCE_DATA/PAM/ 解析缺失问题分析
- PAM 目录虽有 enabled=true + parser=warehouse 配置，但实际解析产出接近零
- 根因1：DDL 单文件（1.2M行/10584表）→ DDLParser.parse_file() 只返回1个 TableInfo
- 根因2：DML 使用 Oracle PL/SQL execute immediate 动态SQL → warehouse DML 正则无法匹配
- 根因3：PAM 用 pas. schema 前缀 → SchemaResolver 不识别（只认 IOL/ITL/IDL 等数仓层级）
- RRP/数仓目录中的 PAMS 数据是不同处理层级的加工，不是重复；PAM 目录才是原始业务系统数据
- 需要新增 Oracle PL/SQL 适配器或改造 warehouse 解析器才能解析 PAM 数据
