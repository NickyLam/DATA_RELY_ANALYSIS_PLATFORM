## 修复：字段加载失败（404）
- **根因**：`loadTableFields` 调用 `/api/tables/{name}/fields`，但实际运行的是 FastAPI (`app.main:app`)，该端点只存在于旧的 `api_server.py`
- **修复**：
  - `selectTable(fullName, searchIdx)` 增加 `searchIdx` 参数，优先从搜索结果的 `columns` 字段直接获取字段列表（零额外请求）
  - `loadTableFields` 改为通过 `/api/tables?keyword=短名` 精确搜索获取 columns（作为回退方案）
  - 修复 `setupSearchListeners` 中 `_fieldInput` → `fieldInput` 变量名 typo

## 修复：字段下拉框三连问题
- **问题1 - 无法下拉出字段**：选中表后 placeholder 显示"从41个字段中选择"，但点击/聚焦输入框时无下拉列表。原因是 `onFieldInput` 只在输入事件触发过滤，无点击/聚焦显示全部字段的逻辑
  - 修复：在 `fieldInput` 上添加 `focus` 事件监听，聚焦时若无输入则调用 `showAllFields()` 显示全部字段（最多50个），有输入则调用 `onFieldInput()` 过滤
- **问题2 - 输入框错位**：CSS 中 `.input-group` 分两处定义（line 659 和 line 772），合并为单一定义确保 `position: relative` 正确应用；下拉框 `max-height` 从 200px 增至 320px
  - 修复：合并 CSS 重复定义，下拉框 z-index 已有 1000 足够
- **问题3 - 默认Tab**：打开页面默认显示"解析层"而非"展示层"
  - 修复：`app.js` 中 `AppState.currentTab` 改为 `'display'`；`index.html` 中 `.active` class 移到展示层 Tab 按钮和内容区；`DOMContentLoaded` 中增加 `initDisplayTab()` 调用
- **修改文件**：`static/js/search-panel.js`、`static/js/app.js`、`static/index.html`、`static/css/style.css`

## 修复：字段下拉框样式与交互（第二轮）
- **根因分析**：
  1. CSS 中 `.input-group` 的 `min-width: 280px` 小于内部 input 的 `min-width: 340px`，导致下拉框（`left:0; right:0`）宽度与 input 不一致
  2. 旧的 `.search-results` 样式（`width: calc(var(--sidebar-width) - 36px); margin-left: 18px`）是旧版侧边栏布局遗留，会干扰新的顶部横向布局
  3. `.search-result-item`、`.result-name`、`.result-type` 各有两组重复定义，合并为单组
  4. `showAllFields` 函数定义在 `setupSearchListeners` 内部且缩进不规范，移到 IIFE 顶层便于其他函数调用
  5. `setupSearchListeners` 可能被多次调用（每次 `initDisplayTab`），增加 `_searchListenersInitialized` 防重复
  6. 浏览器缓存旧 CSS/JS，所有静态资源加 `?v=2.3` 版本号
- **修复内容**：
  - `style.css`：`.input-group` min-width 从 280px → 340px；移除 input 的 min-width；删除旧版 `.search-results` 样式；合并重复 CSS 规则；input 加 `position:relative; z-index:1; background:#ffffff`
  - `search-panel.js`：`showAllFields` 移到 IIFE 顶层；增加 null check
  - `lineage-graph.js`：`setupSearchListeners` 增加防重复初始化保护
  - `index.html`：所有 CSS/JS 引用加 `?v=2.3` 版本号
