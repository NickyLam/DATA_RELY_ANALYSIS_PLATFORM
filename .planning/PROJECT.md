# 数据血缘分析平台 - 项目上下文

## 📋 项目概述

**项目名称**：异构数据库血缘分析平台  
**项目版本**：v1.0（一期）  
**创建日期**：2026-04-18  
**项目状态**：初始化阶段

## 🎯 项目愿景

建设统一的数据血缘分析平台，**一期优先实现 Oracle 19c 数据库的血缘分析能力**，实现对 Oracle 平台的表级、字段级、作业级、批次级血缘统一采集、统一建模、统一管理和统一展示，重点解决"结果表完整加工血缘可查"和"字段最小血缘链路可追"的核心问题。

## 📌 核心痛点

1. **对象级加工链路不透明**：结果表来自哪些上游表、经过哪些中间层表，缺乏统一视图
2. **字段级来源解释困难**：无法快速回答某个指标字段、报送字段的来源与计算路径
3. **异构平台采集难度高**：不同平台 SQL 方言、元数据接口、作业机制存在差异
4. **运行态与设计态脱节**：文档中描述的加工逻辑与实际生产运行可能不一致
5. **排障效率低**：问题排查严重依赖资深人员经验，难以快速定位问题上游

## 🎯 建设目标

### 一期目标（6 个月）

- ✅ 实现关键字段最小血缘链路查询

- 🔜 增强影响分析与变更审计能力
- 🔜 支持人工修正和审核确认流程

### 三期目标
- 🔮 与数据质量平台联动
- 🔮 与调度平台、开发平台、元数据治理平台集成

## 🛠️ 技术栈

### 后端技术栈
- **语言**：Python 3.11+ (LTS 版本)
- **Web 框架**：FastAPI 0.100+
- **ORM**：SQLAlchemy 2.0+
- **任务队列**：Celery 5.3+
- **缓存**：Redis 7.0+

### 前端技术栈
- **框架**：React 18+ (LTS 版本)
- **语言**：TypeScript 5.0+
- **构建工具**：Vite 4+
- **UI 组件库**：Ant Design 5+ / Material UI
- **可视化**：D3.js 7+ / React Flow / Cytoscape.js
- **状态管理**：Zustand / Redux Toolkit

### 数据层
- **元数据存储**：PostgreSQL 15+ (或复用 DataHub 的 MySQL)
- **图数据库**：Neo4j 5.x (LTS 版本)
- **搜索引擎**：Elasticsearch 8.x (可选，用于全文搜索优化)

### 开源组件
- **元数据平台**：DataHub 0.13+ (推荐) 或 OpenMetadata 1.0+
- **SQL 解析引擎**：
  - Oracle/OceanBase: JSqlParser 4.x (通过 JPype 调用)
  - GBase/崖山：Druid SQL Parser 1.2+
  - TDH Inceptor: ANTLR 4.x + 自定义语法
- **容器化**：Docker 24+、Docker Compose 2.20+

### 部署环境
- **容器编排**：Docker Compose (一期) → Kubernetes (二期/三期)
- **应用服务器**：Gunicorn + Uvicorn (ASGI)
- **Web 服务器**：Nginx (反向代理)
- **监控**：Prometheus + Grafana

## 📐 架构设计

### 六层架构

```
┌─────────────────────────────────────────┐
│           展示层 (Presentation)          │
│  资产搜索 | 完整血缘 | 最小链路 | 影响分析  │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│            服务层 (Service)              │
│  元数据服务 | 血缘查询 | 最小链路 | 审核  │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│            存储层 (Storage)              │
│   PostgreSQL (元数据) + Neo4j (图谱)    │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│            建模层 (Modeling)             │
│   表模型 | 字段模型 | 作业模型 | 血缘边   │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│            解析层 (Parsing)              │
│  SQL 方言识别 | AST 解析 | 血缘提取       │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│            采集层 (Collection)           │
│  JDBC 采集 | SQL 采集 | 运行时采集 | 人工  │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│           数据源层 (Data Sources)        │
│ Oracle | OceanBase | GBase | 崖山 | TDH │
└─────────────────────────────────────────┘
```

## 📦 功能模块（一期 Must Have）

### 8.1 资产搜索与定位模块
- 支持按表名、字段名、系统名、schema 查询
- 支持业务域、主题域、监管场景过滤
- 搜索结果展示基础信息、所属系统、更新时间

### 8.2 对象级完整血缘模块
- 展示结果表从贴源层到结果层的完整加工链路
- 支持表视图、表 + 任务视图、全量视图切换
- 支持层级展开、节点拖拽、缩放、导出
- **一期聚焦：Oracle 19c 数据库对象**

### 8.3 字段级最小血缘链路模块
- 从目标字段向上回溯到贴源字段
- 仅保留与该字段相关的必要路径
- 支持展示转换表达式、聚合逻辑、过滤条件

### 8.5 问题排查模块（基础版）
- 展示完整血缘和最小链路
- 关联最近运行批次、运行状态
- 展示最近结构变更、任务变更

### 8.8 采集任务管理模块
- 配置采集源、采集频率
- 手工触发采集、查看采集日志
- 支持告警通知

## ⚠️ 主要风险与控制措施

### 技术风险
- **风险**：异构数据库方言复杂，自动解析准确率不足
- **措施**：建立人工修正与审核机制、建立方言规则库和解析回归测试集

### 性能风险
- **风险**：平台建设初期对象量大，查询性能可能受到影响
- **措施**：建立分层缓存与图查询优化机制、血缘查询结果缓存（Redis）

### 集成风险
- **风险**：TDH 运行链路接入复杂，运行时采集难度较大
- **措施**：优先聚焦高价值场景，逐步扩展、建立全链路监控机制

## 📊 验收标准

### 性能指标
- 简单搜索（精确表名）：P95 ≤1 秒，P99 ≤2 秒
- 表级完整血缘（≤5 层）：P95 ≤3 秒，P99 ≤5 秒
- 字段级最小链路：P95 ≤5 秒，P99 ≤8 秒
- 支持并发用户数：≥200 人同时在线
- 支持并发查询：≥50 QPS

### 容量规划（一期）
- 管理表数量：≤50,000 张
- 管理字段数量：≤500,000 个
- 血缘边数量：≤2,000,000 条
- 图数据库规模：≤10GB

### 解析准确率
- L1 简单场景：准确率≥98%
- L2 中等场景：准确率≥90%
- L3 复杂场景：准确率≥80%
- L4 极复杂场景：准确率≥70%
