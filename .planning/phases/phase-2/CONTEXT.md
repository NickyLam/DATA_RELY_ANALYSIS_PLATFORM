### 语言和框架
- **Python**: 3.11+ (LTS)
- **Java**: JSqlParser (通过 JPype)
- **Celery**: 异步任务调度

## 任务包分解

### WBS-2.1: Oracle 19c 采集适配器增强（4 人周）

#### 子任务
1. **Task 2.1.1**: JDBC 元数据采集器增强
   - 连接池优化
   - 批量采集支持
   - 错误处理增强

2. **Task 2.1.2**: 系统视图采集器优化
   - ALL_TABLES, ALL_TAB_COLUMNS
   - ALL_CONSTRAINTS, ALL_INDEXES
   - ALL_DEPENDENCIES

3. **Task 2.1.3**: 存储过程源码采集
   - ALL_SOURCE 视图采集
   - 函数、包体、触发器
   - 源码解析和存储

4. **Task 2.1.4**: PL/SQL 深度解析
   - 常见场景解析规则
   - 动态 SQL 处理
   - 解析准确率优化

### WBS-2.2: SQL 解析引擎集成（3 人周）

#### 子任务
1. **Task 2.2.1**: JSqlParser 集成
   - JPype 配置
   - JSqlParser 4.x 集成
   - Python-Java 桥接

2. **Task 2.2.2**: Oracle 方言适配
   - Oracle 特有语法支持
   - MERGE, WITH, MODEL 等
   - 方言规则库

3. **Task 2.2.3**: 血缘提取规则
   - INSERT/UPDATE/DELETE 解析
   - SELECT 子查询解析
   - JOIN 关系提取

### WBS-2.3: 运行时血缘采集（2 人周）

#### 子任务
1. **Task 2.3.1**: Oracle 审计日志采集
   - DBA_AUDIT_TRAIL 视图
   - 审计日志解析
   - 执行语句提取

2. **Task 2.3.2**: V$SQL 视图采集
   - 实时 SQL 监控
   - 执行计划采集
   - 绑定变量提取

3. **Task 2.3.3**: 血缘融合
   - 静态血缘 + 运行态血缘
   - 可信度评分
   - 冲突处理

## 验收标准

### 功能验收
- ✅ 试点表采集完整度 ≥ 98%
- ✅ L1 简单 SQL 解析准确率 ≥ 98%
- ✅ L2 中等 SQL 解析准确率 ≥ 90%
- ✅ 采集任务可配置、可监控

### 性能验收
- ✅ 单表采集时间 ≤ 5 秒
- ✅ 批量采集 1000 表 ≤ 30 分钟
- ✅ SQL 解析延迟 ≤ 100ms

## 风险识别

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| Oracle PL/SQL 解析复杂 | 高 | 中 | 建立人工补录机制、回归测试集 |
| 动态 SQL 难以解析 | 高 | 中 | 运行时采集辅助、人工审核 |
| JSqlParser Oracle 方言支持不足 | 中 | 低 | 自定义语法扩展 |
