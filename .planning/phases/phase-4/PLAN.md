## Wave 1: 试点接入 (WBS-4.1)

**执行步骤**：
1. 创建 `backend/app/config/pilot_tables.py` 配置文件

2. 创建试点表初始化脚本
   - 写入 Neo4j 初始化数据
   - 配置表级血缘关系
3. 创建配置验证脚本
   - 验证表配置完整性
   - 验证数据源连接

**文件创建**：
- `backend/scripts/init_pilot_data.py`
- `backend/scripts/verify_pilot_config.py`

**验收标准**：
- ✅ 试点表配置文件创建完成
- ✅ 试点表数据写入 Neo4j
- ✅ 配置验证脚本运行成功

---

### Task 4.1.2: 采集任务执行

**目标**：执行试点表的元数据采集

**执行步骤**：
1. 配置 Oracle 19c 数据源
   - 设置连接参数（host, port, database, username）
   - 配置采集范围（schema, table）
2. 执行元数据采集
   - 调用 `backend/app/collectors/jdbc_collector.py`
   - 调用 `backend/app/collectors/oracle_views_collector.py`
3. 执行 SQL 解析
   - 调用 `backend/app/parsers/sql_parser.py`
   - 调用 `backend/app/parsers/lineage_builder.py`
4. 写入血缘数据
   - 写入 Neo4j 图数据库
   - 写入 PostgreSQL 元数据库

**验收标准**：
- ✅ 试点表元数据采集完成（≥98%）
- ✅ SQL 解析准确率达标（L1≥98%, L2≥90%）
- ✅ 血缘数据写入 Neo4j 成功

---

### Task 4.1.3: 血缘准确性验证

**目标**：验证采集血缘与人工标注的准确性

**执行步骤**：
1. 创建人工标注配置

   - 定义字段级血缘链路
2. 创建验证脚本
   - 对比采集血缘与人工标注

3. 生成验证报告
   - 输出准确率统计
   - 输出差异分析

**文件创建**：
- `backend/scripts/verify_lineage_accuracy.py`
- `docs/pilot-validation-report.md`

**验收标准**：
- ✅ 表级血缘准确率 ≥ 95%
- ✅ 字段级血缘准确率 ≥ 85%
- ✅ 验证报告生成完成

## Wave 2: 性能优化 (WBS-4.2)

### Task 4.2.1: Redis 缓存服务

**目标**：实现查询缓存提升响应速度

**执行步骤**：
1. 创建缓存服务模块
   - 创建 `backend/app/services/cache_service.py`
   - 实现 Redis 连接管理
2. 实现血缘查询缓存
   - 缓存表级血缘查询结果

   - 设置缓存过期策略（5 分钟）
3. 实现搜索结果缓存
   - 缓存资产搜索结果
   - 缓存筛选器选项
4. 创建缓存失效机制

   - 手动清除缓存 API

**文件创建**：
- `backend/app/services/cache_service.py`

**验收标准**：
- ✅ 缓存服务模块创建完成
