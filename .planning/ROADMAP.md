## 阶段分解

### 阶段 1: 平台底座建设 (M1)
**时间**：第 1-2 月  
**状态**：🔴 未开始  
**目标**：完成 DataHub、Neo4j 部署和基础框架搭建

#### 任务包
- [ ] **WBS-1.1**: DataHub 部署与定制开发（2 人周）
  - 部署 DataHub 0.13+
  - 配置 MySQL/PostgreSQL 后端
  - 自定义血缘实体模型
  - 集成 Neo4j 图查询
  
- [ ] **WBS-1.2**: Neo4j 图数据库部署（1 人周）
  - 部署 Neo4j 5.x
  - 设计血缘图谱模型
  - 配置索引和约束
  - 性能调优
  
- [ ] **WBS-1.3**: 基础框架搭建（2 人周）
  - 后端：FastAPI 项目结构
  - 前端：React + TypeScript + Vite
  - Docker Compose 编排配置
  - CI/CD流水线配置

**交付物**：
- DataHub 运行实例
- Neo4j 运行实例
- 前后端项目骨架
- Docker Compose 配置文件

**验收标准**：
- ✅ DataHub API 可正常访问
- ✅ Neo4j Browser 可连接
- ✅ 前后端项目可启动
- ✅ Docker Compose 可一键启动所有服务

## 关键路径

```
WBS-1.1 → WBS-1.2 → WBS-1.3 → WBS-2.1 → WBS-2.2 → WBS-2.4 → 
WBS-3.2 → WBS-3.3 → WBS-4.1 → WBS-4.2 → WBS-4.3
```

**关键路径说明**：
- DataHub 部署（WBS-1.1）是所有采集工作的前提
- TDH 采集器（WBS-2.2）复杂度最高，是阶段 2 的关键
- 字段级最小链路（WBS-3.3）算法复杂度最高，是阶段 3 的关键
- 试点接入（WBS-4.1）直接决定项目能否成功上线
